callback functions

En Callback funktion er en funktion som bliver sendt som et argument i en anden funktion, og som så bliver eksekveret senere.

app.get('/', (req, res) => {

({data: "Welcome to the first server!"});

});

request - response, de to parametre som callbackfunktionen tager imod vi er på serveren - vi sender response

npm

NPM (Node Package Manager) er en pakke manager for Node.js.

Package.json

package.json er en fil der beskriver ens Node.js projekt. det indeholder information om ens applikation, så som navn, version, dependencies og scripts.

For at lave en Package.json fil skal man skrive denne kommando:

npm init

i terminalen i projekt mappen

når man installere en pakke med npm, bliver den tilføjet til og opdaterer package.json automatisk, det kunne f.eks se sådan her ud:

npm install express

hvis du skriver det i terminalen bliver express tilføjet til dit projekt og du vil kunne se dette i din package.json fil:

node_modules

node_modules er en mappe i vores Node projekt, som opbevarer vores externe dependencies. Man må ikke pushe node_modules op til git, fordi det kan give mergekonflikter og rod!. Derfor skal man huske at skrive den i sin gitignore fil.

Client & Server

Client:

Det brugeren ser. Client siden er den som sender requests. browser og frontend (js, html, css)

Server:

Server siden er den der modtager og behandler requests. Backend - Logik og data.

Get requests

En GET request er når clienten laver en request til serveren, så sender vi et svar tilbage.

Det kan se sådan her ud:

app.get('/snowstorms', (req, res) => {

res.send({data: "Warning: snowstorm ahead!"});

});

her sender vi en html fil som svar:

router.get('/', (req, res) => {

res.send(frontpagePage);

});
