Clean Code

Hvad er Clean code, og hvordan opretholder vi det ?

Vi holder clean code ved at holde os til nogle specifikke regler og principper , dem kommer vi ind på herunder:
  • 1. læsbarhed
  • 2. consistensy

    • Linter:

    Definition: en Linter er et værktøj der kan kigge kode igennem for fejl, stilistiske problemer og afvigelser fra kodestandarder. Den kan ikke kører kode men laver derimod en statisk analyse.

    Commit beskeder:

    Man skal huske at skrive gode og beskrivende commit beskeder, så man gør samarbejdet med collaborators nemmere, og så man kan læse på beskederne hvor man er i projektets timeline

    Git i Terminalen

    Git er et versionsstyringssystem, der bruges til at vedligeholde kode og gør det muligt for flere udviklere at arbejde sammen. Det gør det muligt at gemme og tilgå forskellige versioner af ens kode.

    kommandoer:

    git init

    Laver et nyt Git repository i den mappe du står i.

    git clone 'repository-url'

    Henter en kopi a en allerede eksisterende repository fra f.eks. github.

    git commit -m "message"

    Gemmer de nye ændringer til repositoriet med den givne besked.

    git checkout 'branch-navn'

    Skifter til en anden branch

    git pull

    henter og merger ændringer fra en branch fra f.eks github ned til det lokale repository.

    git push

    uploader mit commit til f.eks github

    Node.js

    Hvad er det?

    Hvad er Node.js ? Node.js er et JavaScript runtime miljø, der gør det muligt at kører JavaScript udenfor web browseren. Det kan bruges til at bygge sever-side applikationer, API'er og fullstack applikationer.

    hvordan kører man filer i det?

    For at køre filer i node, skal man være i mappen, hvor filen er, og skrive:

    node app.js

    og herefter enter. og så kører den filen.
